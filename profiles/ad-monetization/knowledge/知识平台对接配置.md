# 内部知识管理平台 API 对接配置

## 用途说明
本文档定义汇报管理助手与内部知识管理平台的API对接方式，供工作流中 httpRequest 节点使用。

---

## API 配置

### 基础信息

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 基础URL | `https://knowledge.internal.example.com/api/v1` | 按实际环境替换 |
| 认证方式 | Bearer Token | 在系统凭据中配置 `knowledge_platform_token` |
| 超时时间 | 30秒 | 单次请求超时 |
| 重试次数 | 2 | 失败后自动重试 |

### 接口清单

#### 1. 获取汇报文档列表
```
GET /documents?category=report&business_line={line}&type={okr|annual}&limit=3&sort=date_desc

Response:
{
  "documents": [
    {
      "id": "doc_001",
      "title": "2025Q3 OKR汇报 - 流量运营",
      "category": "report",
      "business_line": "流量运营",
      "type": "okr",
      "period": "2025-Q3",
      "created_at": "2025-10-15",
      "download_url": "/documents/doc_001/content"
    }
  ]
}
```

#### 2. 获取文档内容
```
GET /documents/{doc_id}/content
Accept: text/markdown

Response: 文档全文（markdown格式）
```

#### 3. 上传文档
```
POST /documents
Content-Type: multipart/form-data

Fields:
  - file: 文档文件（支持 .md, .docx, .pptx, .pdf）
  - category: "report" | "todo" | "template"
  - business_line: 业务线名称
  - type: "okr" | "annual"
  - period: "2025-Q4"
  - tags: ["历史汇报", "OKR"]

Response:
{
  "id": "doc_new",
  "status": "uploaded",
  "message": "文档已上传并入库"
}
```

#### 4. 搜索文档
```
GET /documents/search?q={keyword}&category=report&limit=10

Response:
{
  "results": [
    {
      "id": "doc_001",
      "title": "...",
      "snippet": "...匹配片段...",
      "relevance_score": 0.85
    }
  ]
}
```

## 工作流节点配置示例

### httpRequest 节点 — 拉取历史汇报

```json
{
  "method": "GET",
  "url": "{{ $env.KNOWLEDGE_PLATFORM_URL }}/documents?category=report&business_line={{ $parameter.business_line }}&type={{ $parameter.report_type }}&limit=3&sort=date_desc",
  "headers": {
    "Authorization": "Bearer {{ $env.KNOWLEDGE_PLATFORM_TOKEN }}",
    "Accept": "application/json"
  }
}
```

### httpRequest 节点 — 上传汇报存档

```json
{
  "method": "POST",
  "url": "{{ $env.KNOWLEDGE_PLATFORM_URL }}/documents",
  "headers": {
    "Authorization": "Bearer {{ $env.KNOWLEDGE_PLATFORM_TOKEN }}"
  },
  "body": {
    "content": "{{ $input.item.final_report }}",
    "category": "report",
    "business_line": "{{ $parameter.business_line }}",
    "type": "{{ $parameter.report_type }}",
    "period": "{{ $parameter.period }}",
    "tags": ["历史汇报"]
  }
}
```

## 环境变量配置

在系统环境或 `config/forge.yaml` 中配置：

```yaml
env:
  KNOWLEDGE_PLATFORM_URL: "https://knowledge.internal.example.com/api/v1"
  KNOWLEDGE_PLATFORM_TOKEN: "your-api-token-here"
```

---

## 注意事项

1. **Token 安全**：API Token 不应硬编码在工作流中，通过环境变量或凭据管理获取
2. **降级方案**：如果 API 不可用，工作流应降级为使用本地知识库（RAG）
3. **数据同步**：从平台拉取的文档会自动入库到本地 RAG 知识库，保持双向一致
4. **速率限制**：注意平台 API 的 rate limit，建议单次拉取不超过 10 个文档
