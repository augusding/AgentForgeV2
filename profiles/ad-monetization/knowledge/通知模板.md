# 汇报管理通知模板

## 用途说明
本文档定义汇报管理助手的各类通知模板，供工作流 notification 节点和通讯渠道使用。

---

## 一、TODO催办通知

### 1.1 即将到期提醒（截止前3天）

```
[提醒] 待办事项即将到期

@{责任人}，您有以下待办事项即将到期，请注意安排：

事项：{todo_title}
来源：{business_line} {report_type}汇报（{meeting_date}）
截止日期：{deadline}
剩余时间：{remaining_days}天
验收标准：{acceptance_criteria}

请在截止日期前完成并反馈进展。如需延期请提前说明原因。

---
此消息由汇报管理助手自动发送
```

### 1.2 超期催办通知

```
[催办] 待办事项已超期

@{责任人}，以下待办事项已超过截止日期，请立即处理：

事项：{todo_title}
来源：{business_line} {report_type}汇报（{meeting_date}）
原定截止：{deadline}
已超期：{overdue_days}天
优先级：{priority}

请于 {new_deadline} 前完成，并回复：
1. 当前进展
2. 预计完成时间
3. 如需变更方案请说明原因

如连续3天未回复，将升级至上级领导。

---
此消息由汇报管理助手自动发送
```

### 1.3 升级通知（超期未响应）

```
[升级] 待办事项超期未处理

@{manager}，以下待办事项已超期且责任人未及时响应：

事项：{todo_title}
责任人：{assignee}
原定截止：{deadline}
已超期：{overdue_days}天
催办次数：{reminder_count}次
最后催办日期：{last_reminder_date}

建议处理方式：
- 与责任人确认阻碍原因
- 调整截止日期或变更方案
- 指定替代人选

---
此消息由汇报管理助手自动发送
```

## 二、TODO建档通知

### 2.1 新建档通知（发给所有责任人）

```
[通知] 汇报待办事项已建档

各位同事，以下待办事项已从 {report_type}汇报（{meeting_date}）中提取并建档：

共 {total_count} 条待办：
{todo_summary_table}

各位请确认自己负责的事项，如有问题请及时反馈。
系统将在截止前3天自动提醒，超期自动催办。

---
此消息由汇报管理助手自动发送
```

## 通知渠道适配

| 渠道 | 状态 | 格式要求 |
|------|------|---------|
| 系统内通知 | 已支持 | Markdown格式，通过 notification 节点 log 通道 |
| 邮件 | 预留 | HTML格式，通过 emailSmtp 节点 |
| 内部通讯工具 | 预留 | 按内部API格式适配，通过 httpRequest 节点 |

### 降级策略
当外部通知渠道不可用时，所有通知自动降级为系统内日志通知，确保信息不丢失。
